# SecureChat

Безопасное веб-приложение для обмена сообщениями в реальном времени с использованием криптографии AES-GCM.

## Описание

SecureChat - это полнофункциональное чат-приложение, которое обеспечивает:

- **Шифрование сообщений** с использованием AES-GCM криптографии
- **Защита от атак** (XSS, CSRF, MIME sniffing)
- **Аутентификация пользователей** с безопасными токенами
- **Реал-тайм обмен сообщениями** между пользователями
- **История сообщений** с сохранением в JSON файлы

## Технологический стек

### Backend
- **Python 3**
- **Flask** - веб-фреймворк
- **Flask-CORS** - поддержка CORS
- **cryptography** - библиотека для шифрования

### Frontend
- **HTML5** - структура приложения
- **CSS3** - стилизация
- **JavaScript (Vanilla)** - интерактивность

## Установка

### Требования
- Python 3.7+
- pip (менеджер пакетов Python)

### Шаги установки

1. Клонируйте репозиторий:
```bash
git clone https://github.com/MagicRaven01/SecureChat.git
cd SecureChat
```

2. Установите зависимости:
```bash
pip install flask flask-cors cryptography
```

3. Запустите сервер:
```bash
python chat_server.py
```

4. Откройте браузер и перейдите на:
```
http://localhost:5000
```

## Использование

1. **Регистрация**: Используются только ID (генерируются рандомно при каждом новом открытии чата)
2. **Чат**: Отправляйте сообщения другим пользователям
3. **Безопасность**: Все сообщения автоматически шифруются перед отправкой и ключи тоже

## Возможности безопасности

### Шифрование
- Сообщения шифруются с использованием AES-256-GCM
- Уникальные ключи шифрования для каждого пользователя
- Нонсы (IV) для каждого сообщения

### Защита от атак
- **Content Security Policy (CSP)** - защита от XSS атак
- **X-Frame-Options** - предотвращение встраивания в фреймы
- **X-Content-Type-Options** - защита от MIME type sniffing
- **CORS** - контроль доступа между доменами
- **Безопасные токены сессии** - защита сессий пользователя

## API Endpoints

- `POST /register` - Регистрация нового пользователя
- `POST /login` - Вход пользователя
- `POST /send_message` - Отправка сообщения
- `GET /get_messages/<recipient>` - Получение сообщений
- `GET /` - Главная страница приложения
